## 1. 기존 사이트 분석

기존 사이트(thebgyeong.headstone01.gethompy.com)는 단일 페이지 세로 스크롤 구조로 구성되어 있으며, 12개 객실 정보가 세로로 길게 나열되어 있어 모바일 사용자의 탐색 피로도가 높다. 내비게이션은 우측 상단 햄버거 메뉴 하나로만 접근 가능하며, 스페셜 항목은 슬라이드 형태로 제공되나 인터랙션이 제한적이다. 예약 및 인스타그램 링크가 하단에 위치하여 사용자가 원하는 시점에 빠르게 접근하기 어렵다는 구조적 한계가 있다.

| 항목 | 현황 | 개선 방향 |
|------|------|-----------|
| 레이아웃 | 세로 나열형 단일 페이지 | 섹션별 스와이프 카드 UI |
| 객실 탐색 | 12개 세로 스크롤 | 타입 탭 필터 + 가로 스와이프 카드 |
| 스페셜 | 단순 이미지 슬라이드 | 카드 클릭 시 바텀 시트 상세 정보 |
| 예약 접근 | 하단 고정 | 스티키 하단 바 (항상 노출) |
| 인스타 연동 | 상단 아이콘 링크 | GNB + 스티키 바 + 하단 섹션 |

## 3. 페이지 구조 및 UX 흐름

### 3.1. 인트로 페이지

실제 더 비경 사진 3장이 풀스크린으로 페이드 전환되며, 중앙에 로고(logo_circle.png)와 **입장하기** 버튼이 배치된다. 하단에는 슬라이드 인디케이터(점)가 표시되며, 8초 후 자동으로 메인 페이지로 전환된다. 버튼 클릭 시 즉시 전환된다.

### 3.2. 메인 페이지 구조

```
[GNB] 더 비경 로고 | 인스타 아이콘 | 햄버거 메뉴
  ↓
[Hero] 풀스크린 슬라이드 + 뱃지 + 제목 + CTA 버튼 2개
  ↓
[객실 섹션] 타입 탭 필터 (전체/풀빌라/스파빌라/프라이빗빌라) + 가로 스와이프 카드
  ↓
[스페셜 섹션] 감성 이미지 카드 스와이프 → 클릭 시 바텀 시트 상세
  ↓
[혜택 배너] 공식 예약 혜택 (최대 10% 할인 + 웰컴 기프트)
  ↓
[퀵 액션] 예약 | 인스타그램
  ↓
[브랜드 스토리] 감성 카피 문구
  ↓
[인스타 프리뷰] 3x2 그리드
  ↓
[Footer] 사업자 정보
  ↓
[스티키 하단 바] 예약하기 | 인스타 (항상 노출)
```

### 3.3. 전체 메뉴 (오버레이)

햄버거 메뉴 클릭 시 다크 오버레이로 전체 메뉴가 표시된다. 메뉴 항목은 **객실 / 스페셜 / 혜택 / 예약 / 인스타그램** 5개로 구성되며, 각 항목에 한글명과 영문명이 함께 표기된다.

---

## 4. 주요 인터랙션 설계

### 4.1. 객실 타입 탭 필터
전체/풀빌라/스파빌라/프라이빗빌라 탭 클릭 시 해당 타입의 객실 카드만 표시되며, 카드 목록이 맨 처음으로 스크롤된다.

### 4.2. 스페셜 바텀 시트
스페셜 카드 클릭 시 화면 하단에서 바텀 시트가 슬라이드업 되며, 핵심 포인트 / 이용 안내 / 추천 포인트 / CTA 버튼이 구조화되어 표시된다. 배경 클릭 또는 X 버튼으로 닫힌다.

### 4.3. 스티키 하단 바
메인 페이지 진입 후 항상 화면 하단에 고정되어 **예약하기** 버튼과 **인스타** 버튼이 노출된다. 예약 전환율(Conversion Rate) 극대화를 위한 핵심 UI 요소이다.

## 7. API 연동 계획 (digitalnow.co.kr)

GNB 구성 및 모든 설명/정보 텍스트는 `digitalnow.co.kr` API와 연동하는 구조로 설계한다. 목업 단계에서는 정적 데이터로 구현하였으며, 실제 개발 시 다음 항목을 API로 대체한다.

- GNB 메뉴 항목 및 링크
- 객실별 이름, 설명, 가격, 이미지 경로
- 스페셜 항목별 이름, 설명, 이용 안내
- 혜택 배너 텍스트
